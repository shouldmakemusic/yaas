RemixNet.UDPServer

This class is a barebones UDP server setup with the ability to assign callbacks for incoming data. In the design as is, we use an OSC.CallbackManager when we recieve any data.

This class is designed to be used by RemixNet.OSCServer, as it will do all the setup for you and register a few default OSCManager callbacks.

source code 

Sets up the UDPServer component of this package. By default we listen to all interfaces on port 9000 for incoming requests with a 4096 byte buffer.

You can modify these settings by using the methods setport() and setHost()

processIncomingUDP(self)

source code 

Attempt to process incoming packets in the network buffer. If none are available it will return. If there is data, and a callback manager has been defined we'll send the data to the callback manager.

You can specify a callback manager using the UDPServer.setCallbackManager() function and passing it a populated OSC.Manager object.

setCallbackManager(self, callbackManager)

source code 

You can specify a callbackManager here as derived from OSC.py. We use this function in OSCServer to register the default /remix/ namespace addresses as utility callbacks.

bind(self)

source code 

After initializing you must UDPServer.listen() to bind to the socket and accept whatever packets are in the buffer. Since we're binding a non-blocking socket, your program (and Ableton Live) will still be able to run.
